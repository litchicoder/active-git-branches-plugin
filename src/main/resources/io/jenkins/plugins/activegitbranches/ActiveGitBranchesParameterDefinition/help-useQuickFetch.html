Controls the fallback behavior when no workspace is available.

Fetch Strategy (in order of preference):

  1. Local refs (instant): Reads cached refs from existing .git directory. No network required, instant response, time-sorted.
  2. Workspace fetch: If no local refs, fetches from remote with 30s timeout.
  3. Fallback (when no workspace):
    • Enabled (default): Uses ls-remote. Very fast (~1s), alphabetical sort.
    • Disabled: Uses shallow clone. Slow (10+s), time-sorted.

Tip: Run a build first to populate local refs, then "Build with Parameters" will be instant with time-based sorting.
